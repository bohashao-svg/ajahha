一、基础信息

1. 项目名称：RunningHub iOS客户端

2. 运行环境：iOS 15.0+

3. 开发模式：本地代码编写 + Gitlab Actions云端自动构建IPA，无需本地Xcode编译

4. 核心集成：SS_tools 鸭鸭图v1.2解码工具（https://github.com/copyangle/SS_tools）

二、核心功能要求

1. 支持输入工作流ID/链接，自动提取ID、拉取云端工作流

2. 自动识别工作流类型：文生图 / 文生视频 / 图生视频

3. Plus模式独立开关：默认开启，可手动关闭；开启携带mode:plus，关闭走普通模式

4. 自动检测工作流是否包含duck_encode_node v1.2，参数完全对齐工作流原生节点

5. 内置鸭鸭图解码工具，有鸭鸭图节点生成的结果自动解码鸭子图

6. 云端实时并发校验，提交按钮自动置灰/自动恢复

7. 有并发数时提交任务无阻塞，可连续提交新任务

8. 右上角全局任务中心，支持进度查看、预览、相册保存

9. 所有数据本地加密持久化，重启不丢失

三、核心硬性规则

1. 并发规则：仅以RunningHub云端配额为准，并发位释放自动刷新按钮状态

2. 提交规则：仅修改用户手动填写参数，不篡改原始工作流结构

四、业务主流程

1. 首次启动 → 录入RunningHub API密钥，本地加密保存

2. 输入工作流ID/链接 → 自动提取ID → 拉取工作流JSON

3. 自动识别工作流类型 + 检测鸭鸭图v1.2节点

4. 动态渲染表单：基础参数 + Plus开关 + 鸭鸭图专属参数（有节点才显示）

5. 用户填写参数 → 自动校验并发资格

6. 提交任务（按Plus开关携带对应参数）→ 主界面立即重置，支持连续提交

7. 后台轮询任务状态，完成后自动分支处理：

◦ 鸭鸭图任务：自动解码 → 详情页展示原图，支持保存原图/保存加密鸭子图

◦ 普通任务：直接展示成品，无需解码

8. 任务结束自动停止轮询，并发空闲自动恢复提交按钮

五、页面结构

1. 首页：ID输入框、拉取按钮、Plus开关、参数表单、并发状态提示、提交按钮、右上角任务入口

2. 任务中心：4个标签（排队中/生成中/已完成/失败），任务卡片标注鸭鸭图/Plus标识

3. 任务详情：进度展示、媒体预览、重新解码按钮、双保存按钮、取消/重试功能

4. 设置页：修改API密钥、Plus全局默认设置、清除历史记录

六、关键接口清单

1. 查询用户并发配额：GET /v1/user/quota

2. 获取工作流详情：GET /v1/workflow/detail

3. 提交生成任务：POST /v1/workflow/run

4. 批量查询任务状态：GET /v1/task/batch

5. 取消任务：POST /v1/task/{task_id}/cancel

七、异常处理

1. 网络异常：友好提示，自动重试

2. API密钥失效：弹窗引导重新配置

3. 解码失败：展示鸭子图，支持手动重输密码解码

4. 并发已满：按钮自动置灰，空闲自动恢复

5. 多设备操作：全部以云端数据同步为准

八、外部依赖

1. SS_tools官方v1.2解码网页（https://github.com/copyangle/SS_tools）

2. RunningHub官方开放API

八、appUI设计：
1.禁止一眼Ai风格（避开渐变紫）
2.使用不规则的卡片式布局
3.配色简约
4.图标采用svg（禁止使用emoji）
